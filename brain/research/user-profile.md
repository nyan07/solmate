# User Research — Target Profile

## Who

- Age 16+, core focus **18–40**
- Spend money in cafés/bars/restaurants regularly
- Live in **urban areas in Northern Europe** — Germany, Scandinavia, Netherlands
- Cities with dense gastronomy infrastructure where tall buildings block the sun

## Usability Test Script (August 2025)

### Scenario prompt

> "You're about to leave work and decide it's a good evening for a drink on a sunny terrace. This app should help you find the right spot."
> Think aloud: what do you see, where do you go, what do you expect?

### Interview questions

**Understand/validate the problem**

1. How often do you go out for coffee, drinks, or food?
2. How do you choose where to go?
3. What main factors influence your venue decision? _(probe: sun/weather preferences)_
4. Have you ever specifically looked for a sunny spot? If so, how?
5. When would you prefer shade?
6. How far in advance do you typically plan?

**Solution validation**

1. How appealing is an app that shows you sunny spots?
2. What features would it need?
3. What information would help you decide for/against a place?
4. When would you use this type of app?
5. Real-time (next 30 min) or plan ahead — which do you prefer?

### Assumptions being tested

1. `Find` — User can navigate the map to locate a spot
2. `Find` — User uses the list to narrow down the search
3. `Find` — Detail card info is useful for final decision
4. `Time` — User wants sunny spots now or within next few hours
5. `Decision` — User expects weather to be included in prediction
6. English UI is acceptable

## Beta Testing (launched November 2025)

### Goal

Test market fit, usability, and performance.

Success criteria:

1. Arkie has all features the user considers valuable for regular use
2. Tester finds sunny spots via list and map view
3. Info helps identify the best spot for their next drink
4. No major bugs or performance issues

- 10–15 testers, friends & family, living in cities
